格式

格式

下面是Dockerfile的格式

# Comment
INSTRUCTION arguments

这些指令是不区分大小写的。但是为了能和参数更容易的区分，一般使用大写。

Docker按顺序执行Dockerfile中的命令。为了指定你构建的基础镜像第一条指令必须为'FROM'。

Docker把以#开头的行作为注释，除非这一行是一个有效的解析器指令。一个#在这一行的非开头位置将会被当做参数。他允许类似于下面的语句：

# Comment
RUN echo 'we are running some # of cool things'

连续的命令当中夹杂注释是不被允许的。
